# Pastel React Starter

Minimal React starter scaffolded with Vite 8 тАФ themed with the Pastel color system.

## Stack

- **Vite 8** + **React 19** + **TypeScript**
- **React Compiler** (`babel-plugin-react-compiler`)
- **code-inspector-plugin** тАФ Alt + click any DOM node to open its source in Cursor
- **vite-plugin-route-builder** тАФ file-based routes from `src/pages/`
- **TailwindCSS v4** + **@pastel-palette/tailwindcss** theme
- **next-themes** for light/dark
- **jotai** for state
- **@lobehub/eslint-config** + **@lobehub/prettier-config**

## Use this template

```sh
pnpm dlx degit Innei/Pastel/templates/react-starter my-app
cd my-app
pnpm i
pnpm dev
```

## Project layout

```
src/
тФЬтФА components/ui/        UI primitives (no business knowledge)
тФЬтФА features/<domain>/    Business modules тАФ components, atoms, hooks, barrel
тФЬтФА pages/                File-based routes тАФ every .tsx exports `Component`
тФЬтФА lib/                  Cross-feature pure utilities
тФЬтФА styles/globals.css    Tailwind + Pastel theme + dark variant
тФЬтФА providers.tsx         ThemeProvider + Jotai Provider
тФФтФА App.tsx               Mounts routes inside providers
```

Path alias: `~` тЖТ `./src`.

## Routing

Add a file under `src/pages/` and it becomes a route automatically.

- `pages/foo.tsx` тЖТ `/foo`
- `pages/blog/[id].tsx` тЖТ `/blog/:id`
- `pages/(group)/layout.tsx` тЖТ wraps grouped routes
- Add `.sync.tsx` to opt out of lazy loading

Each page file must export a named `Component` (and optional `loader`). See [vite-plugin-route-builder](https://www.npmjs.com/package/vite-plugin-route-builder).

`src/generated-routes.ts` is auto-regenerated by the plugin at `vite dev` / `vite build`. It ships with `// @ts-nocheck` and is also excluded from ESLint тАФ commit changes when the route surface evolves.

## Theming

Light/dark is driven by `next-themes` via `data-theme="dark"`. The mapping to Tailwind's `dark:` variant lives at the top of `src/styles/globals.css`:

```css
@custom-variant dark (&:where([data-theme=dark], [data-theme=dark] *));
```

Use semantic Pastel classes тАФ `bg-background`, `text-text`, `border-border`, `bg-accent` тАФ and they switch automatically. See the [Pastel docs](https://github.com/Innei/Pastel) for the full token list.

## State

`jotai` provider is wired in `providers.tsx`. Define atoms inside the relevant `features/<domain>/atoms.ts` and consume with `useAtom`.

## Editor jump

Hold **Alt** and click any DOM node in the browser тАФ the file opens in Cursor at the right line. Configure `editor` in `vite.config.ts` for other editors (`vscode`, `webstorm`, etc.).
