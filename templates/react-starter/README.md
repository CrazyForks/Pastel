# Pastel React Starter

Minimal React starter scaffolded with Vite 8 — themed with the Pastel color system.

## Stack

- **Vite 8** + **React 19** + **TypeScript**
- **React Compiler** (`babel-plugin-react-compiler`)
- **Base UI** (`@base-ui/react`) — headless primitives behind the bundled UI kit
- **motion** via `LazyMotion` (`domAnimation` + `strict`, ~6KB instead of ~30KB)
- **code-inspector-plugin** — Alt + click any DOM node to open its source in Cursor
- **vite-plugin-route-builder** — file-based routes from `src/pages/`
- **TailwindCSS v4** + **@pastel-palette/tailwindcss** theme
- **next-themes** for light/dark
- **jotai** for state
- **@lobehub/eslint-config** + **@lobehub/prettier-config**

## Use this template

```sh
pnpm dlx degit Innei/Pastel/templates/react-starter my-app
cd my-app
pnpm i
pnpm dev
```

## Project layout

```
src/
├─ components/ui/        UI primitives (no business knowledge)
├─ features/<domain>/    Business modules — components, atoms, hooks, barrel
├─ pages/                File-based routes — every .tsx exports `Component`
├─ lib/                  Cross-feature pure utilities
├─ styles/globals.css    Tailwind + Pastel theme + dark variant
├─ providers.tsx         ThemeProvider + Jotai Provider
└─ App.tsx               Mounts routes inside providers
```

Path alias: `~` → `./src`.

See [AGENTS.md](./AGENTS.md) for the full file-placement and convention guide (also wired into `CLAUDE.md` for AI coding agents).

## UI components

`src/components/ui/` ships a small kit built on Base UI and styled with Pastel semantic tokens: button, input/textarea, form, modal, dropdown menu, context menu, select, popover, tooltip, toast (sonner). The home page (`src/pages/index.tsx`) demos every one of them.

Modal and context menu also expose imperative APIs (hosts are mounted in `providers.tsx`):

```tsx
import { confirmModal, createModal } from '~/components/ui/modal'
import { showContextMenu } from '~/components/ui/context-menu'

createModal({ title: 'Hi', content: <Body /> })
confirmModal({ title: 'Delete?', danger: true, onOk: async () => api.delete() })
onContextMenu={(e) => { e.preventDefault(); showContextMenu(items) }}
```

## Routing

Add a file under `src/pages/` and it becomes a route automatically.

- `pages/foo.tsx` → `/foo`
- `pages/blog/[id].tsx` → `/blog/:id`
- `pages/(group)/layout.tsx` → wraps grouped routes
- Add `.sync.tsx` to opt out of lazy loading

Each page file must export a named `Component` (and optional `loader`). See [vite-plugin-route-builder](https://www.npmjs.com/package/vite-plugin-route-builder).

`src/generated-routes.ts` is auto-regenerated by the plugin at `vite dev` / `vite build`. It ships with `// @ts-nocheck` and is also excluded from ESLint — commit changes when the route surface evolves.

## Theming

Light/dark is driven by `next-themes` via `data-theme="dark"`. The mapping to Tailwind's `dark:` variant lives at the top of `src/styles/globals.css`:

```css
@custom-variant dark (&:where([data-theme=dark], [data-theme=dark] *));
```

Use semantic Pastel classes — `bg-background`, `text-text`, `border-border`, `bg-accent` — and they switch automatically. See the [Pastel docs](https://github.com/Innei/Pastel) for the full token list.

## State

`jotai` provider is wired in `providers.tsx`. Define atoms inside the relevant `features/<domain>/atoms.ts` and consume with `useAtom`.

## Editor jump

Hold **Alt** and click any DOM node in the browser — the file opens in Cursor at the right line. Configure `editor` in `vite.config.ts` for other editors (`vscode`, `webstorm`, etc.).
